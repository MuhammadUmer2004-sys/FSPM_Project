# 🚀 Freelance-Flow: Current Project Features

Use this list for your presentation and project documentation. These features were specifically chosen to demonstrate **Software Project Management (FSPM)** principles.

---

### 1. 🛡️ FSPM Management Features (The "Marks Earners")
These features prove you understand planning, scoping, and estimation.

*   **W5HH Project Definition**: The "Post Task" form follows the **W5HH principle** (Why, What, When, Who, Where, How, How much). This ensures every project has a clear business justification before it starts.
*   **Explicit Scope Boundaries**: The UI has hard-coded sections for **"In-Scope"** vs. **"Out-of-Scope."** This prevents **Scope Creep** (the #1 killer of software projects) by forcing clients to list exclusions (like payment processing) explicitly.
*   **COCOMO-Lite Estimation Tool**: An integrated calculator that uses basic COCOMO logic (Number of Features × Complexity Multiplier) to suggest a realistic budget to the client. This shows you understand **Software Cost Estimation**.
*   **WBS Phased Bidding**: Developers don't just bid a total price; they break their bid into **Phases/Milestones** (e.g., Design, Development, Testing). This is a direct application of the **Work Breakdown Structure (WBS)**.
*   **Audit Trail Logs**: Every bid submission is automatically timestamped and logged in the database. This ensures **Process Integrity** and transparency, which is critical for SPM.

---

### 2. 💻 Core Functional Features (The "App" Logic)
These make the platform work as a usable marketplace.

*   **Role-Based Access Control (RBAC)**: Distinct user experiences for **Clients** (who post and manage projects) and **Developers** (who browse and submit bids).
*   **Real-Time Bidding Dashboard**: A central area where clients can view and compare all incoming bids for their projects in one view.
*   **W5HH Descriptions**: Project descriptions are saved in a structured format, making them professional and easy for developers to evaluate.
*   **Milestone Tracking**: The ability to see exactly how a developer plans to execute a project through their submitted phases.

---

### 3. ⚙️ Technical Implementation (The "Full Stack" Build)
How the system is actually built.

*   **Frontend**: Built with **React + Vite** for a blazing-fast, modern UI.
*   **Backend**: Powered by **Node.js + Express** to handle API requests efficiently.
*   **Database**: **MongoDB (NoSQL)** database used to store complex nested data like milestone arrays and scoping rules.
*   **Aesthetics**: Premium **Dark Mode & Glassmorphism** design using Vanilla CSS (Internal CSS) for a state-of-the-art look.
*   **Connectivity**: Ready for **MongoDB Atlas** (Cloud) for team collaboration or **MongoDB Compass** (Local) for testing.
